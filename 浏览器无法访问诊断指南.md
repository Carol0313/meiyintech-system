# 浏览器无法访问诊断指南

## 当前服务器状态

服务器实际已在运行，curl 测试通过：
- `http://127.0.0.1:8000/login/` → 返回 200 ✅
- 端口 8000 正在监听 ✅

如果浏览器仍然无法打开，请按以下顺序排查：

---

## 排查步骤

### 1. 确认访问的 URL 完全正确

浏览器地址栏必须输入：
```
http://127.0.0.1:8000/login/
```

**注意：**
- 必须是 `http://`，不是 `https://`
- 末尾有斜杠 `/`
- 不要用 `localhost`，用 `127.0.0.1`

### 2. 清除浏览器缓存

按 `Ctrl + Shift + R` 强制刷新页面

或按 `Ctrl + Shift + Delete` 清除缓存后重试

### 3. 换一个浏览器测试

尝试用以下浏览器访问：
- Chrome
- Edge
- Firefox

### 4. 检查防火墙/安全软件

某些安全软件可能会阻止对 127.0.0.1:8000 的访问：
- 360安全卫士
- 腾讯电脑管家
- Windows Defender

尝试暂时关闭防火墙测试：
1. 打开"Windows 安全中心"
2. 点击"防火墙和网络保护"
3. 暂时关闭"专用网络防火墙"

### 5. 使用启动脚本重新启动

双击运行项目目录下的 `start_server.bat`，它会：
1. 关闭旧的服务器进程
2. 重新启动服务器
3. 自动测试并打开浏览器

### 6. 检查浏览器控制台错误

按 `F12` 打开开发者工具，查看 Console 标签页：
- 如果有红色错误，截图发给我
- 查看 Network 标签页，看请求是否被拦截

---

## 备用访问方式

如果以上都不行，可以尝试：

### 方式一：使用本机 IP 地址
```
http://192.168.1.x:8000/login/
```
（将 192.168.1.x 替换为您的实际局域网 IP）

### 方式二：更换端口号

修改 `start_server.bat`，将 `8000` 改为 `8080` 或 `9000`
