# CHANGELOG 管理 Skill

## 用途

自动维护 `CHANGELOG.md` 变更日志文件。每次用户要求进行代码修改、功能新增、Bug修复等变更时，自动记录到变更日志中。

## 触发条件

当用户要求进行以下操作时，必须更新 CHANGELOG：
- 新增功能（feat）
- 修复Bug（fix）
- 修改样式/UI（style）
- 重构代码（refactor）
- 性能优化（perf）
- 数据库变更（data）
- 部署/配置变更（chore）
- 文档更新（docs）

## 更新流程

### 1. 读取当前 CHANGELOG

```bash
cat CHANGELOG.md
```

### 2. 确定变更类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | 新增红框修改功能 |
| `fix` | 修复 | 修复订单金额计算错误 |
| `style` | 样式/UI | 修改侧边栏颜色 |
| `refactor` | 重构 | 重构订单查询逻辑 |
| `perf` | 性能优化 | 优化拼版算法 |
| `chore` | 构建/部署 | 配置Nginx |
| `data` | 数据库 | 新增字段、迁移数据 |
| `docs` | 文档 | 更新README |

### 3. 确定日期

使用当前日期（ISO格式：YYYY-MM-DD）

### 4. 编写变更条目

格式：
```markdown
### N. 变更标题 [类型]
- **功能/改动**：简要描述
- **影响文件**：文件路径
- **备注**：其他说明
```

### 5. 更新文件

将新条目插入到对应日期下（按时间倒序，最新的在前面）

### 6. 提交并推送

```bash
git add CHANGELOG.md
git commit -m "docs: 更新CHANGELOG.md - [变更摘要]"
git push origin master
```

## 变更条目模板

```markdown
### N. 变更标题 [类型]
- **功能/改动**：具体描述做了什么
- **影响文件**：`文件路径1`, `文件路径2`
- **数据库**：是否有迁移（是/否，迁移文件名称）
- **备注**：其他需要记录的信息
```

## 示例

```markdown
## 2026-06-06

### 1. 红框尺寸修改功能 [feat]
- **功能**：商户后台可修改红框尺寸，支持添加/删除框
- **影响文件**：`apps/merchant_platform/views.py`, `templates/merchant/order_detail.html`
- **数据库**：否
- **备注**：修改后自动重新计算订单金额

### 2. 修复订单金额计算错误 [fix]
- **改动**：修复了面积计算时单位转换错误
- **影响文件**：`apps/orders/models.py`
- **数据库**：否
- **备注**：影响所有已支付订单
```

## 注意事项

1. **每次变更都必须记录**，不要遗漏
2. **变更描述要清晰**，让团队成员能快速理解做了什么
3. **影响文件要列出**，方便后续追溯
4. **数据库变更要特别标注**，提醒需要执行迁移
5. **推送失败要重试**，确保变更日志同步到GitHub
6. **如果当天已有条目**，在已有日期下追加，不要新建日期

## 文件位置

`CHANGELOG.md` 位于项目根目录
