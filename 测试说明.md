# 镁印制版下单系统 - 内部测试说明

> 本文档面向公司内部测试人员，说明如何在不安装任何开发环境的情况下参与系统测试。

---

## 一、基本原理

系统部署在一台**主机电脑**上作为服务器，其他同事的电脑只需通过浏览器访问主机的局域网地址即可，**不需要安装 Python、虚拟环境或任何软件**。

---

## 二、主机电脑操作（由管理员执行）

### 1. 启动服务器

在项目文件夹内，双击运行：

```
start_server.bat
```

启动成功后，命令行窗口会显示：

```
服务器已启动，请用浏览器访问：
http://127.0.0.1:8000/login/
```

> ⚠️ **不要关闭这个窗口**，关闭即停止服务。

### 2. 确认主机局域网 IP

启动服务器后，在命令行中查看本机内网 IP（或让管理员告知）。

示例：`192.168.1.18`

### 3. 防火墙设置（重要）

如果同事无法访问，请执行以下任一操作：

**方式A：临时关闭防火墙（仅测试期间）**
以管理员身份运行 CMD，输入：
```bash
netsh advfirewall set allprofiles state off
```

**方式B：仅开放 8000 端口（推荐）**
以管理员身份运行 CMD，输入：
```bash
netsh advfirewall firewall add rule name="Django Dev Server" dir=in action=allow protocol=tcp localport=8000
```

## 三、测试人员操作（其他同事）

### 1. 打开浏览器

推荐使用 **Chrome** 或 **Edge** 浏览器。

### 2. 访问系统

在地址栏输入主机电脑的局域网地址，例如：

```
http://192.168.1.18:8000/
```

> 请将 `192.168.1.18` 替换为实际的主机 IP。

商家后台支持创建**岗位子账号**，不同岗位拥有不同权限：

### 创建子账号的方法

1. 用商家管理员（13800138000）登录商家后台

3. 点击"新增子账号"，填写手机号、密码、选择岗位角色
4. 保存后即可用新手机号登录测试

## 六、常见问题

### Q1：浏览器提示"无法访问此网站"
- 检查主机电脑是否已启动 `start_server.bat`
- 确认输入的 IP 地址和端口是否正确（例如 `192.168.1.18:8000`）
- 检查主机防火墙是否已开放 8000 端口
- 确认主机和测试电脑连接在同一个局域网（同一个 WiFi / 路由器）

### Q2：页面样式错乱或白屏
- 按 `Ctrl + F5` 强制刷新
- 换用 Chrome 或 Edge 浏览器

### Q3：上传文件失败
- 检查文件大小是否超过限制
- 确认网络连接稳定

### Q4：登录提示"账号或密码错误"
- 确认使用的是正确的平台和对应账号
- 注意区分总平台、商家后台、客户平台的登录入口

### Q5：主机电脑关机后还能访问吗？
- **不能**。这是开发测试服务器，主机电脑关机或关闭命令行窗口后服务即停止。
